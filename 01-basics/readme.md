## 🧠 Core Terms

### Document
A single record in MongoDB, stored in JSON-like format.

```json
{
  "name": "Seema",
  "age": 30
}
```

---

### Collection
A group of documents — similar to a table in SQL.
Example: `users`, `orders`, `products`

---

### BSON
Stands for **Binary JSON** — MongoDB's internal storage format.
Faster than JSON and supports more data types (Date, ObjectId etc.)

---

### Database
A container for collections.
Example: `myAppDB`, `ecommerce`

---

### Field
A key-value pair inside a document.
Example: `"name": "Seema"`

---

### _id
Unique identifier for every document. Auto-generated by MongoDB using **ObjectId**.
→ See [object-id.md](./object-id.md) for full breakdown.

---

### Schema
Structure of data. In MongoDB it is **flexible** — no fixed columns required.

---

### Index
Speeds up query performance — like an index in a book.
→ Will be covered in `04-indexing/`

---

### Query
A request to fetch data from a collection.

```js
db.users.find({ age: 30 })
```

---

### Aggregation
Used for data processing — group, filter, transform results.
Similar to SQL `GROUP BY`.
→ Will be covered in `03-aggregation/`

---

### Sharding
Splitting data across multiple servers for **horizontal scaling**.

---

### Replication
Copying data across multiple servers for **high availability and backup**.

---

### MongoDB Atlas
Cloud-hosted version of MongoDB — no local setup required.
Used for production deployments.

---
